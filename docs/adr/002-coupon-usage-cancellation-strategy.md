# ADR-002: クーポン使用記録のキャンセル戦略

## ステータス

採用

## コンテキスト

クーポン機能において、注文がキャンセルされた場合にクーポン使用記録（coupon_uses）をどう扱うかを決める必要があった。

仕様上の制約：
- 1ユーザーにつき同じクーポンは1回まで使用可能
- キャンセル時はクーポンを未使用に戻し、再利用可能にする

## 検討した案

### 案A: レコード削除

キャンセル時に coupon_uses のレコードを削除する。

- メリット: シンプル。(coupon_id, user_id) の通常のユニーク制約で1回制限を保証できる
- デメリット: 使用履歴が消える。「過去にこのクーポンが使われてキャンセルされた」という事実が追えなくなる

### 案B: ステータス管理

coupon_uses に status カラム（used / cancelled）を持たせる。キャンセル時は status を cancelled に変更する。

- メリット: 履歴が残る。キャンセルされた使用記録も追跡可能
- デメリット: cancelled レコードが残るため、単純な (coupon_id, user_id) のユニーク制約では再利用時に制約違反になる。部分ユニークインデックスが必要になり、複雑さが増す

## 決定

**案Aを採用。** キャンセル時はレコードを削除し、(coupon_id, user_id) の単純なユニーク制約で1回制限を保証する。

## 採用理由

- ステータス管理にすると部分ユニークインデックスが必要になり、設計が必要以上に複雑になる
- 最小構成で始めて後から拡張する方針に合う
- 履歴管理が必要になった場合は、クーポンに限らず全体的な監査ログの仕組みとして別途検討する
