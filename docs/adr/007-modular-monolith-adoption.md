# ADR-007: モジュラーモノリスへの移行

## ステータス

提案中

## コンテキスト

`TODO.md` の Phase 2 としてモジュラーモノリス化が計画されているが未着手。Phase 1（EC MVP）が完了し、Phase 3（レビュー・在庫）に進む前に移行へ着手するかを判断する。

「まだコードの複雑さが足りず、モジュラ化の効果を体感しにくいのではないか」という懸念があり、移行時期が論点となっていた。

## 検討した案

### 案A: 今、モジュラーモノリスへ移行する

### 案B: 複雑さが増してから移行する（Phase 3 以降）

- メリット: モジュラ化前後のギャップが大きく、効果を体感しやすい

## 決定

**案Aを採用。**

## 採用理由

- `monolith` ブランチに現状を残せば、移行後も旧状態を参照・見比べでき、案 B のメリット（複雑化からの移行体験）は A でも代替できる

## 影響

- 移行前に `monolith` ブランチを作成して現状のモノリス版を保存する。以降 `monolith` ブランチには原則機能追従しない
- Phase 2 の実装が始動する
- モジュール境界・通信規約・イベント基盤は後続 ADR（ADR-008 以降）で個別に決定する
