# カート + 注文機能 設計メモ

## 今回のゴール

この会話で決めること:

---

## 1. ユーザーストーリー

すべてログイン必須。

- カートに商品を追加できる
- カートを閲覧できる
- カートから商品を削除できる
- カートの商品の個数を変更できる
- 注文を確定できる
- 注文をキャンセルできる（いつでも可能）
- 注文履歴を一覧で見られる
- 注文の詳細を見られる

## 2. 機能仕様

### カート

- 同一商品はカート内で1行にまとまる（重複行にならない）
- 商品追加時に数量が増える
- 数量の上限なし（在庫管理導入時に見直す）
- 出品者に削除された商品は「この商品は削除されました」と表示し、注文対象から除外
- カートはDB（バックエンド）で保持する
- 細かいUI挙動は実装時に決める

### 注文

- 注文確定時にカートは空になる
- ステータスは「確定済み」「キャンセル済み」の2つ
- キャンセル時はステータス変更のみ（カートに商品は戻さない）
- 注文時の価格を注文側に記録する（後から商品価格が変わっても注文履歴は変わらない）
- 注文履歴の表示項目: 注文番号、注文日時、商品名・個数・単価、合計金額、ステータス、商品へのリンク

## 3. 機能別設計（DB・API・画面）

### カートに商品を追加する
- DB: カートテーブル、カートアイテムテーブル（なければ作成、同じ商品があれば個数+1、なければ新規行追加）
- API: POST /api/v1/cart/items（商品IDを受け取る）
- 画面: 商品一覧or詳細ページに「カートに追加」ボタン

### カートを閲覧する
- DB: カート+カートアイテム+商品テーブルを読む
- API: GET /api/v1/cart
- 画面: カート画面

### カートから商品を削除する
- DB: カートアイテムの該当行を削除
- API: DELETE /api/v1/cart/items/:item_id
- 画面: カート画面に削除ボタン

### カートの商品の個数を変更する
- DB: カートアイテムの個数を更新
- API: PATCH /api/v1/cart/items/:item_id（数量を送る）
- 画面: カート画面に+/-ボタン。数量1のときは-ボタン非活性

### 注文を確定する
- DB: 注文テーブル+注文アイテムテーブルに書き込み、カートアイテムを削除
- API: POST /api/v1/orders
- 画面: カート画面に「注文確定」ボタン

### 注文をキャンセルする
- DB: 注文テーブルのステータスを「キャンセル済み」に更新（レコードは削除しない）
- API: PATCH /api/v1/orders/:id/cancel
- 画面: 注文詳細ページに「キャンセル」ボタン

### 注文履歴を一覧で見る
- DB: 注文テーブルをユーザーIDで取得
- API: GET /api/v1/orders
- 画面: 注文履歴一覧ページ

### 注文の詳細を見る
- DB: 注文テーブル+注文アイテムテーブルを結合して取得
- API: GET /api/v1/orders/:id
- 画面: 注文詳細ページ（商品名・個数・単価・合計金額・ステータス・商品リンク）

## 4. データモデル

※正規化は後で行う

### 注文テーブル
- 注文番号（UUID）
- ユーザーID
- 注文日時
- ステータス

### 注文アイテムテーブル
- 注文ID
- 商品ID
- 商品名（商品削除後も履歴に表示するため）
- 単価（購入時の価格）
- 個数

### カートテーブル
- ユーザーID（1ユーザー1カート）

### カートアイテムテーブル
- カートID
- 商品ID
- 個数

※PKはRails標準のid、created_at/updated_atはRailsに任せる
